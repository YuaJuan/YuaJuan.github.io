fabric-ca的使用

Posted on 2018-07-19 Edited on 2019-09-10 In 区块链 , Hyperledger Fabric

两种和hyperledger fabric 交互的方式：

  1. 通过hyperledger fabric ca client
  2. 通过fabric sdk
    这两种方式都是通过rest api交互的。

    See fabric-ca/swagger/swagger-fabric-ca.json for the swagger documentation for these REST APIs. You may view this documentation via the http://editor2.swagger.io online editor.(这个链接不能访问？)

Hyperledger fabric ca client可能链接到hyperledger fabric ca的集群上。那么HA Proxy会考虑负载均衡选择一个fabric-ca-server用于服务。

在集群中的所有hyperledger fabric ca 服务器都共享通一个数据库来记录identties和certificates. 如果LDAP配置了，那么identity information 会被保存在LDAP中，而不再是数据库中。

一个服务器可能包含多个ca。每个ca要么是个根ca或者一个中间ca。每个中间ca都有一个父ca。

1.Fabric CA Server

启动server前需要先初始化，会产生一个默认的配置文件可以reviewed以及customized在启动服务之前。

Fabric CA server 的根目录决定如下：

-home command > FABRIC_CA_SERVER_HOME>FABRIC_CA_HOME>CA_CFG_PATH>当前目录

目前假设配置了FABRIC_CA_HOME=$HOME/fabric-ca/server,并且server configuration文件已经存在于服务器根目录

1.2启动服务器

  • 如何不存在ca-cert.pem and ca-key.pem files，那么会重新生成。同样，如何配置文件不存在，也会重新生成。

  • 通过设置tls.enabled to true,那么服务将会监听在https,而不是http

  • 设置registry.maxenrollments,可以控制同一个enrollment ID的密码被使用的最大次数。

  • 服务默认监听在7054端口
