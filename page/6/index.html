docker--网络配置

Posted on 2019-09-13 Edited on 2019-09-10 In docker

Docker的网络子系统是可插拔的（pluggble),使用drivers。几个drivers是默认存在的，并且提供了以下几个核心的网络功能：

  • brigde: 默认的网络驱动模式。如果你的应用运行在一个独立的容器里面，与之通信常用bridge networks。或者多个运行在同一主机上的容器需要通信，brigde也是最好的选择。

  • host: 对于独立的容器，去掉容器和docker宿主机上的隔离，并且直接使用宿主机的网络。host模式只使用于Docker 17.06及更高版本的swarm服务中。

  • overlay: overlay网络把多个docker daemons连接到一起并且使得swarm中的docker可以相互通信。也可以使用overlay网络让一个独立运行的容器和swarm中的容器通信。（有多个运行在不同主机上的容器需要通信，最好用overlay)

Posted on 2019-09-13 Edited on 2019-02-22 In docker

启动与配置参数

  1. 网络启动过程
    Docker 服务启动时会首先在主机上自动创建一个 dockerO虚拟网桥， 实际上是一个Linux 网桥。 网桥可以理解为一个软件交换机， 负责挂载其上的接口之间进行包转发。
    同时， Docker 随机分配一个本地未占用的私有网段（在 RFC1918 中定义）中的一个地址
    给 dockerO接口。 比如典型的 172.17.0.0/16 网段， 掩码为 255.255.0.0。 此后启动的容器内的网口也会自动分配一个该网段的地址。
