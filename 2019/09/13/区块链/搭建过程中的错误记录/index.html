Hyperledger Fabric 错误记录

Posted on 2019-09-13 Edited on 2019-02-22 In 区块链 , Hyperledger Fabric

panic: Unable to bootstrap orderer. Error reading genesis block file: read /var/hyperledger/orderer/orderer.genesis.block: is a directory

change orderer.genesis.block to genesis.block

在cli容器中执行peer channel create -o orderer.example.com:7050 -c $CHANNEL_NAME -f ./channel-artifacts/channel.tx

transport: http2Client.notifyError got notified that the client transport was broken unexpected EOF.
Error: could not send: rpc error: code = Unavailable desc = transport: write tcp 172.20.0.16:50250->172.20.0.15:7050: write: connection reset by peer

查明在配置文件中ORDERER_GENERAL_TLS_ENABLED=true
所以这里需要执行tls的路径

ORDERER_CA=/opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem
peer channel create -o orderer.example.com:7050 -c $CHANNEL_NAME -f ./channel-artifacts/channel.tx –tls –cafile $ORDERER_CA

./bin/configtxgen -profile OrgsChannel -outputAnchorPeersUpdate ./channel-artifacts/Org1MSPanchors.tx -channelID ${CHANNEL_NAME} -asOrg Org1MSP
CRIT 003 Error on inspectChannelCreateTx: Must specify an organization to update the anchor peer for
可能是CHANNEL_NAME的值并没有指定
CHANNEL_NAME=mychannel

更新锚节点时出错
Error: got unexpected status: FORBIDDEN – Failed to reach implicit threshold of 1 sub-policies, required 1 remaining: permission denied
但是更新组织1的节点时没有出错，更新组织2和3时出错，因为最开始环境变量设置的是peer0.org1.example.com的值，所以考虑是环境变量出错。
这项设置错误

CORE_PEER_MSPCONFIGPATH=/opt/gopath/src/github.com/hyperledger/fabric/peer/crypto/peerOrganizations/org${2}.example.com/users/Admin@org${2}.example.com/msp

创建channel出错

Error: failed to create deliver client: orderer client failed to connect to orderer.example.com:7050: failed to create new connection: context deadline exceeded

日志中：
orderer.example.com | panic: Unable to bootstrap orderer. Error reading genesis block file: read /var/hyperledger/orderer/orderer.genesis.block: is a directory

创世块有问题，检查channel-artifacts下是否有正确的genesis.block
注意：如果发现genesis.block是个文件，那说明没有正确的genesis.block。这个文件是在docker挂载时生成的文件。
